# 平台文案规范

## 小红书

### 定位
短平快信息图帖，视觉驱动，适合碎片化阅读。数据和硬信息入图，正文只做钩子。

### 规范
- **正文字数**：≤200字（硬上限300字，超过用户直接跳过）
- **配图尺寸**：9:16（1080×1920）
- **配图数量**：2-5张（根据信息量决定）
- **图片风格**：按内容类型选择（深色科技风/清新插画风）
- **标签**：10-15个，混合热门+精准

### 正文结构
```
[钩子句：1-2句制造好奇/冲突/反差]
[核心观点：用最少的字说清楚]
[引导看图：提示翻图看详细数据]
[互动引导：提问/邀请评论]
[标签]
```

### 配图结构（典型）
- 第1张：封面图（大字标题+核心数据/金句）
- 第2-3张：数据卡/对比卡/要点卡（承载所有硬信息）
- 第4张：总结/金句/互动引导（可选）

## 视频号

### 定位
极简短文案，信息密度高但字数极少。配图同小红书可复用。

### 规范
- **正文字数**：≤100字（硬上限150字）
- **配图尺寸**：9:16（1080×1920）—— 与小红书通用
- **风格**：精简为核心数据+一句话结论+互动引导

### 正文结构
```
[核心信息：1-2句话说清楚是什么]
[关键数据/对比]
[互动引导]
[标签]
```

---

## 微信公众号

### 定位
深度分析长文，面向关注科技/AI的读者。

### 规范
- **字数**：3000-5000字
- **配图**：8张左右，信息图+全景+对比图
- **封面尺寸**：2.35:1
- **结构**：标题→一句话概括→逐个拆解→产业影响→观点总结

### 结构模板
```markdown
# [标题：15字以内]

> 编辑 / [署名]

## 一句话概括
[核心观点，加粗]

## 核心内容：逐个拆解
### 1. [要点名称]：[一句话定位]
[正文：数据+分析+引用]

## 产业影响
[格局分析、竞争对手对比]

## 我们的看法
[观点，信号+问号并列]
```

## 多平台分发策略

同一事件 → 不同角度 → 不同平台：

| 平台 | 策略 |
|------|------|
| 小红书 | 拆分为短帖，每帖聚焦一个亮点，视觉驱动 |
| 视频号 | 与小红书配图通用，文案极简 |
| 公众号 | 全面深度分析，面向专业读者 |

不要简单地把长文缩短变成短帖，而是重新从短帖视角组织信息。
