# 团队分工与协作

## 角色分配

| 角色 | 职责 | 建议模型 |
|------|------|---------|
| 协调者 | 任务理解、流程编排、质量把关、综合判断、用户沟通、小改动直接微调 | Claude Opus 4 |
| 审核者 | 内容审核、运营评估、选题判断、质量把控 | GPT-5 系列（与协调者不同模型） |
| 内容创作者 | 文案撰写+配图关键词设计（四段式 Prompt）、改稿、创意策划 | Claude Opus 4 |
| 执行者 | 资料收集、出图执行+轮询、文件操作、格式整理 | DeepSeek V4 Pro |
| 出图引擎 | 图片生成 | Codex CLI -m gpt-5.5 |

## 核心设计原则

1. **审核者与内容创作者使用不同模型**——形成多角度交叉验证
2. **创意和执行分离**——内容创作者只做创意，执行者只做执行
3. **协调者不亲自跑长时间任务**——出图轮询交执行者，避免占主上下文

## 协作流程

### 选题阶段
1. 协调者或执行者收集资料，形成结构化素材汇总
2. 协调者提出选题方案
3. 审核者对选题方案评估（热度/可操作性/风险）
4. 协调者综合确定最终选题和平台分配

### 创作阶段
1. 内容创作者根据素材+平台规范产出文案+配图关键词（一步完成）
2. 审核者审核（事实准确性、标题吸引力、信息入图合理性）
3. 协调者综合判断（同意/追加意见/请求用户介入）
4. 改稿：小改动协调者直接改，大改动退回内容创作者

### 多篇并行（⭐ 重要）
- 多篇同类任务时，**必须同时开多个内容创作者会话（每人一篇）并行撰写**
- 禁止单个创作者串行赶多篇
- 原因：效率更高 + 隔离性好，避免多任务信息互串

### 出图阶段
1. 协调者做关键词最终检查（四段式合规性）
2. 出图执行交执行者（独立会话）：
   - 写 prompt 文件 → codex 读取生图
   - ≤5 路 background 并行
   - 文件大小判定 + 失败重试
3. **文本渲染检查**：出图完成后用视觉模型逐张检查图上文字正确性
4. 执行者汇报结果给协调者验收

### 发布阶段
1. 协调者验收全部文案和配图
2. 用户审核确认
3. 用户手动发布

## 改稿职责划分

| 改动类型 | 谁来改 | 示例 |
|----------|--------|------|
| **小改动** | 协调者直接改 | 改一两个词、调事实表述、修正数据、微调关键词描述 |
| **大改动** | 退回内容创作者 | 重写段落、大幅调整配图方案、增减图片数量、改变风格方向 |

协调者的角色是**检查+微调**，不是代写者。大量创作性工作始终留在内容创作者手中。

## 角色边界（硬性）

| ❌ 禁止 | ✅ 正确 |
|---------|---------|
| 执行者写文案/做 Prompt 设计 | 内容创作者写 |
| 协调者自己跑出图轮询 | 交执行者 |
| 内容创作者做文件整理/脚本执行 | 交执行者 |
| 协调者既审核又决策 | 审核者审核 → 协调者综合判断 |
| 单个创作者串行赶多篇 | 多个创作者并行 |

## 审核链路

```
内容创作者(步骤6) → 审核者(步骤7) → 协调者综合判断(步骤8) → 改稿(步骤9)
```

审核时对照调研资料检查事实准确性，协调者做最终决策。
