# 阿里云服务器全流程：代操作手册

本 skill 把 agent 定位为**代操作管家**，不是文档朗读机。原则：

> **能自动做的自动做；做不了的给链接手把手教；每个自动步骤都有手动兜底；每一步都说清楚在干什么；花钱和实名的事必须先问过用户。**

用户被假定为**纯小白**：不会命令行、不懂云计算术语。所有面向用户的话术都要用大白话，术语第一次出现时用一句话解释。

## 执行协议（每次使用本 skill 必须遵守）

### 1. 透明播报规则

每个操作步骤，按「做前说、做完报」执行：

- **做前**：一句话说明「现在要做什么 + 为什么」。例：
  > 「接下来我帮你查一下杭州地域 2核2G 服务器的价格，这样你能先知道要花多少钱再决定。」
- **做完**：报结果 + 下一步。例：
  > 「查到了：包年约 XXX 元。下一步是正式下单，下单会从你的阿里云账户扣钱，需要你确认。」
- 长流程（部署环境等）中间每完成一个里程碑播报一次，不要闷头连做十步。
- 失败时如实说明失败原因，**禁止**掩盖失败或不告知就换路径。

### 2. 关键步骤同意清单（做之前必须获得用户明确同意）

| 动作 | 为什么要同意 |
|------|-------------|
| 任何**产生费用**的操作：ECS 下单/续费/升配、域名注册、购买证书、开快照策略等 | 花的是用户的钱；下单前必须先报出**具体价格**（用询价 API 或活动页价格）再问 |
| 引导用户**创建/提供 AccessKey** | 这是账户最高级别凭证之一，用户有权知道风险 |
| **修改安全组**（开放端口） | 影响服务器安全面 |
| **释放/删除任何资源**（实例、云盘、解析记录等） | 不可逆或难恢复 |
| **正式部署/覆盖**服务器上已有内容 | 可能覆盖用户已有数据 |
| **域名解析切换**到新服务器 | 直接影响线上访问 |
| 引导用户**提交备案** | 涉及实名信息和政府审核 |

同意必须是**明确的**（用户说"可以/确认/买吧"），沉默、含糊不算同意。询问时把「要做什么、花多少钱、有什么后果」一次说清。

### 3. Failover（自动失败转手动）规则

- 每个自动步骤失败后：先看报错，能修的（参数错、缺权限提示明确）修一次重试，**最多重试 2 次**。
- 仍失败 → 立即向用户说明「自动操作没成功，原因是 X，我来带你手动做」，切到该步骤的**手动兜底路径**（各 reference 文件中每步都给了）。
- 手动路径 = 给出准确的控制台链接 + 逐步点什么填什么 + 用户做完后 agent 用查询 API/命令**验证结果**（能验证的都要验证，不能验证的让用户截图或口头确认）。
- 用户没有 AccessKey 或不愿提供 → 全流程直接走手动指引模式（本 skill 的所有 reference 都自带完整手动路径，等同 v1 教程）。

### 4. 凭证与安全红线

- AccessKey 建议用户创建**RAM 子用户 AK**并只授予所需权限（AliyunECSFullAccess、AliyunDNSFullAccess 等），不要用主账号 AK。指引见 `references/02-account-prep.md`。
- **绝不**把 AK Secret 打印到聊天记录/日志/文件中明文展示；配置进 CLI 后即闭口不提。
- 不代替用户做实名认证、人脸核验、支付密码类操作——这些是身份行为，只能本人做。
- 服务器登录私钥（.pem）妥善保存并提醒用户备份，聊天中不展示私钥内容。

## 自动化能力边界总表（已联网核实，2026-07）

| 环节 | agent 能自动做 | 必须用户本人做 |
|------|---------------|---------------|
| 选型 | 全部（对话决策 + 询价 API 报价） | — |
| 账号准备 | 在用户电脑上安装/配置阿里云 CLI；验证 AK 有效性 | 注册账号、实名认证（人脸）、创建 AccessKey（控制台点选）、账户充值 |
| 购买 ECS | 查库存/规格（DescribeAvailableResource）、询价（DescribePrice）、创建密钥对/安全组、**下单**（RunInstances，含包年包月，从账户余额/优惠券扣款）、查状态 | 若走**活动页特惠价**（如新人 99 元套餐）→ 只能控制台下单，用户点击支付；余额不足时充值 |
| 部署 | 云助手 RunCommand（免 SSH 全自动执行命令）、或本机 SSH 直连执行全部部署命令、上传代码 | 无（Workbench 网页终端作为用户手动兜底） |
| 域名与解析 | 查域名可注册性（CheckDomain）、DNS 解析增删（AddDomainRecord）、HTTPS 证书自动签发（certbot via RunCommand/SSH） | 域名信息模板实名认证；域名购买建议控制台手动（API 可行但需已实名模板+余额，对新手不划算） |
| ICP 备案 | ❌ 无任何 API，全程无法自动化 | 全部：填表、上传证件、App 人脸核验、工信部短信核验 |

**关于浏览器自动化**：阿里云控制台登录有短信验证/风控，且涉及支付页面。**不要**用浏览器自动化替用户登录控制台或点支付——不可靠且越权。浏览器自动化仅可用于打开公开页面（活动页、文档页）给用户看。自动操作一律走 CLI/OpenAPI。

## 标准主流程

```
需求对话（选型结论）
  → 账号准备（注册/实名 = 人工；CLI+AK 配置 = 半自动）
  → 询价并报价【需同意】→ 下单购买【需同意】
  → （内地服务器）立即启动域名购买+备案（人工，审核 1~3 周）
      ‖ 并行：部署环境（自动）→ 用 IP 调试
  → 备案通过 → 解析域名（自动）【切换需同意】→ HTTPS（自动）→ 上线检查
```

关键节奏提醒：备案审核期间网站不能通过域名访问，但可以用公网 IP 调试——所以「买完服务器当天提交备案，审核期间同步部署」是最省时间的路径。

## 快速决策速查

- **产品线**：个人博客/学习 → 轻量应用服务器；小程序后端/企业应用/要成长空间 → ECS；只有自己跑模型推理才需要 GPU（调 API 不需要）。
- **地域**（买后不可改）：用户在内地 → 内地地域（须备案）；不想备案/海外用户 → 中国香港/海外（免备案，内地访问稍慢）。
- **计费**：建站 → 包年包月（备案也要求包月≥3个月）；临时测试 → 按量付费。
- **新手基线**：2核2G / 3M 固定带宽 / ESSD 40G / Ubuntu LTS；带数据库建议 2核4G。
- **价格**：涉及活动价（如新人特惠）一律以官网活动页当前展示为准，agent 报 API 询价结果时注明是"标准价，活动页可能更便宜"。
